## Anti-Patterns Verdict

**LLM assessment**: Product routes now read as a calm planning desk rather than editorial chrome. The workspace leads with "What to do next" before a demoted readiness summary; itinerary and budget cards are flat. No hero-metric template, gradient text, glass, or icon-card grid.

**Deterministic scan**: `detect.mjs --json` over `src/components/workspace`, `src/components/itinerary`, `src/components/budget`, and `globals.css` returned `[]` (clean, exit 0). The prior snapshot's `/workspace` 27 overlays (22 nested cards, 5 all-caps body) and `/budget` nested cards are resolved at the source level: section kickers demoted, nested `Card`-in-`Card` removed, edit modes flattened to row expansions.

**Visual + a11y evidence**: Browser inspection of all three routes confirmed flat cards, a single accessibility nav set plus footer, a skip link, real trip-derived itinerary logistics, and a real "Last updated" timestamp. Mock collaborators ("Sarah Mitchell", "SM/JW/+2") and the fabricated "Hoshinoya Kyoto" logistics are gone.

## What's Working

- Workspace task flow: compact header, next actions first, readiness as a compact summary row, first incomplete section auto-opens.
- Flattened surfaces: itinerary activity cards expand inline in edit mode; budget insights collapse into one divided column with Value as a header row.
- Honest data: itinerary logistics and timestamps derive from the trip; budget travelers are read-only with a link to trip settings.

## Priority Issues

- **[P2] Help and documentation is thin**: Scoring rules beyond the readiness formula (conflict penalties, value weighting) have no contextual explanation. *Fix*: add small `<details>` helpers next to conflict and value scores. *Suggested command*: `$impeccable clarify`.
- **[P3] Next-action copy grammar**: "2 packing item are still unchecked" should pluralize. *Fix*: pluralize in `getTripNextActions`. *Suggested command*: `$impeccable clarify`.
- **[P3] Destructive deletes rely on undo only**: acceptable with the undo affordance, but a confirm on multi-item deletes would prevent slips. *Suggested command*: `$impeccable harden`.

## Persona Red Flags

**Casey (Distracted Mobile User)**: Next action is now reachable near the top after a one-line header; state persists locally across interruptions. No fixed bottom CTA, so a long header still costs a short scroll.

**Sam (Accessibility-Dependent User)**: Skip link added; inactive nav is `inert`/`aria-hidden`, so the tree exposes one nav set plus the footer. Success status now uses ink text on a positive tint for readable contrast. Disclosure summaries carry a longer accessible name (title plus summary line).

## Minor Observations

- Motion is now section-level only; repeated list rows no longer animate on entrance.
- Product routes share one padding scale (`py-12 lg:py-16`).

## Questions to Consider

- Should conflict and value scoring expose their rules inline, or stay in a single "how scoring works" page?
- Is a mobile sticky next-action CTA worth the collision risk with the bottom tab bar?
